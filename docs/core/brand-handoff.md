# 1. Important Update: the brand name has changed

The company should no longer be referred to externally as **iHouse**.

That name is no longer the public-facing brand and should not be used in outward brand, product, website, marketing, or presentation work.

The brand is now:

**Domaniqo**

Primary domain:

**domaniqo.com**

Important rule:

**Do not return to iHouse as the external name.**

If needed, iHouse may remain only as a temporary internal codename in legacy references, but it should not appear as the external brand.

### ⚠️ Hard Branding Boundary — inside vs. outside

This is a strict, non-negotiable rule:

| Scope | Brand | Examples |
|-------|-------|----------|
| **Internal (inside)** | **iHouse Core** | File names, folder names, module names, service names, router names, env vars, logger names, OpenAPI internal title, ZIP archives, imports, docstrings, technical docs, test files, construction-log, phase-timeline |
| **External (outside)** | **Domaniqo** | UI screens, login page, sidebar brand, external headings, PDFs sent to clients, emails, marketing copy, graphic assets, any product-facing or client-facing surface |

**Do NOT rename any internal system artifact to Domaniqo.**
No file renames, no module renames, no import renames, no env var renames, no logger renames.
The internal system is and remains **iHouse Core**.

Only surfaces visible to end users, guests, property owners, or external stakeholders use **Domaniqo**.

# 17. Motion language

Default transition: 280ms ease-out
Micro interaction: 150ms ease
Entrance reveal: 500ms ease-out, staggered 80ms
Exit / dismiss: 200ms ease-in

Motion follows escalation severity:

Info (booking arrived, sync complete): gentle fade-in, 400ms
Warning (task approaching SLA, prep needed): fade-in + subtle pulse once, 300ms
Critical (ACK_SLA_BREACH, sync failure): immediate appear + persistent dot, no animation loop

Do:
ease-out for entrances
ease-in for exits
stagger children 60–100ms
opacity always accompanies transform
reduce motion for accessibility

Don't:
bounce or spring
elastic overshoot
spinning or rotating
parallax scrolling
decorative animation loops

# 18. Sound identity

Sound principles:
acoustic over synthetic
short (under 1s for actions)
natural materials: wood, bell, piano
one sound per event, never stacked
volume 40–60% of system max
user can mute all sounds

## Sound palette

**Arrival** — new booking created: single soft piano note, C major, 0.4s decay

**Complete** — task done, check-in confirmed: two ascending notes C→E, wooden marimba, 0.5s total

**Attention** — warning, task approaching SLA: single muted bell, slightly longer sustain, 0.6s

**Alert** — ACK_SLA_BREACH, sync failure, critical anomaly: two descending notes G→D, warmer timbre, 0.7s

**Resolve** — anomaly resolved, sync restored: three ascending notes C→E→G, soft pad, 0.8s

**Ambient** — app open, login success: very soft pad chord, C major 7th, 1.5s fade

## Escalation sound map

BOOKING_CREATED → Arrival (Info)
Task assigned → Silent
Task acknowledged → Complete (Confirm)
Task approaching SLA → Attention (Warning)
ACK_SLA_BREACH → Alert (Critical)
COMPLETION_SLA_BREACH → Alert (Critical)
Sync failure → Alert (Critical)
Anomaly resolved → Resolve (Recovery)
BOOKING_CANCELED → Silent

Avoid:
chiptune or 8-bit
notification ding-dong
voice or speech
musical jingles
repeating alarm loops
generic system beeps

# 19. Error and empty states

Empty and error states must feel intentional and calm — like a prepared room before the guest arrives.

### Tone rules

Empty states: welcoming, spacious, unhurried, factual, never apologetic
Warnings: honest, direct, show last known state
Errors: transparent about uncertainty, urgent but controlled
SLA breaches: command voice, not panic

### Key states

Empty dashboard: "Your operations start here." → Add your first property
No reservations: "No stays scheduled yet." → Connect a booking channel
Search empty: "Nothing matches this filter." → Clear filters
Sync failure: "Sync interrupted. Last successful: 2h ago." → Retry sync / View log
Rate conflict: "Rate discrepancy detected. Confidence: partial." → Review details / Override
SLA breach: "Task overdue. Escalation triggered." → Reassign / Acknowledge now

### System states

404 page: monogram + "Page not found." + "Back to Dashboard" — premium, calm, simple
Maintenance: monogram + "Scheduled maintenance. Your data is safe." — reassuring, brief

# 20. Data visualization

### Chart color sequence
Deep Moss (primary) → Quiet Olive (secondary) → Muted Sage (tertiary) → Soft Sand (neutral) → Signal Copper (highlight) → Warm Alert (threshold breach)

### Financial confidence indicator
Every financial chart must show confidence level:

**Confirmed** — direct from provider, verified (Deep Moss, full bar)
**Calculated** — system-derived, high confidence (Quiet Olive, 75%)
**Estimated** — partial data, use with caution (Soft Sand, reduced bar)

### Chart rules
Rounded corners (4px) on bars
No 3D effects
Grid lines in Stone Mist only
Labels in Manrope 10px
Values in Inter Medium
Max 4 series per chart
No pie charts (use horizontal bars)
No gradient fills
No drop shadows on data
No animated number counters
Always show confidence level for financials

# 21. Internationalization

The system currently operates in three languages:

**English** (en) — LTR — Primary brand language
**Thai** (th) — LTR — Characters are wider, allow 20–30% text expansion
**Hebrew** (he) — RTL — Entire layout mirrors

More languages will be added over time.

### Typography per language

Headlines:
English: Manrope SemiBold
Thai: Noto Sans Thai SemiBold
Hebrew: Heebo SemiBold

UI and body:
English: Inter Regular / Medium
Thai: IBM Plex Sans Thai
Hebrew: Assistant Regular

Editorial accent:
English: Instrument Serif
Thai: Noto Serif Thai
Hebrew: Frank Ruhl Libre

Line height:
English and Hebrew: 1.5–1.6
Thai: 1.7–1.8 (taller characters)

All font alternatives are from Google Fonts.

### RTL rules for Hebrew

These elements mirror:
text alignment
navigation order
sidebar position
directional icons (arrows)
card layouts
form labels
padding and margin (swap left and right)

These elements stay fixed and do not mirror:
Domaniqo logo and wordmark (always LTR)
numbers within text
phone numbers
timestamps
progress bars
charts and graphs
code and technical IDs

### Brand name rule
"Domaniqo" is always written in Latin script regardless of UI language.
It does not get transliterated.
Taglines and support lines do get translated per language.

### Text expansion rule
Design all UI containers for the longest language version.
If it works in Thai, it works everywhere.
Thai expands 20–30% from English.
Hebrew expands 5–15%.
Always test buttons, labels, navigation items, and status badges at Thai string length.
