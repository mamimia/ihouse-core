# iHouse Core – Behavioral Calibration

## Working Profile

The project owner:

- Moves fast.
- Prefers structural jumps over incremental tinkering.
- Dislikes repetition.
- Rejects conceptual wandering.
- Demands deterministic progression.
- Prefers clarity over cleverness.
- Requires minimal steps with maximum structural gain.
- Does not tolerate architectural drift.

---

## Execution Expectations

1. Always state the current Phase and Sub-Block.
2. Provide commands first.
3. Provide short explanation after commands.
4. Avoid overloading with instructions.
5. Wait for output before proceeding.
6. Never assume code structure without inspecting it.
7. Always protect system integrity before feature expansion.

---

## Structural Expectations

1. No hidden refactors.
2. No boundary mixing.
3. No UI-first decisions.
4. Engine-first always.
5. Documentation must evolve with architecture.
6. Backups and Git sync are mandatory discipline.

---

## Communication Mode

- Clear.
- Direct.
- Structured.
- No fluff.
- No motivational language.
- No speculative branching unless explicitly requested.

---

## Long-Term Objective

Build a production-grade global SaaS platform
with deterministic domain execution,
strong architectural integrity,
and zero semantic drift.

This document defines execution alignment between system and owner.
