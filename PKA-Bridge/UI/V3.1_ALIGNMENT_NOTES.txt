Property Image → Property Name → Location → Details → Availability

SUBMITTER V3.1 SPECIFICS:

Role: Booking request submission flow
Personality: Guided, trustworthy, progressive disclosure
Visual Confidence: Structured form with clear progress, moss/copper balance

5 Screens:
1. Booking Details (STEP 1 of 3) - Guest name, email, phone, requests
2. Email Verification (STEP 2 of 3) - 6-digit code verification
3. Review & Submit (STEP 3 of 3) - Full review card, checkbox, submit
4. Confirmation Success - Success state with booking confirmation
5. My Bookings (Shared Destination) - Booking status & next steps

Key Patterns:
- Property Context Bar: Top of every form screen (thumbnail + brief info)
  Shows guest is booking THIS property, not generic form
- Progress Bar: Visible, fills as user advances
- Form Labels: Uppercase 11px Manrope 700 (consistent with screen labels)
- Stepper Controls: For dates & guest counts (accessible, clear)
- Review Card: Full booking summary before submit (builds confidence)
- Checkpoint: Email verification ensures deliverable lead
- Success State: Clear confirmation + next steps messaging

Hierarchy:
Property Context → Form Title & Progress → Form Fields → Review → Submit

CONNECTED SYSTEM TREATMENT:

1. COLOR CONSISTENCY:
   Both use #B56E45 (copper) for primary CTAs and accents
   Both use #334036 (moss) for secondary/trusted actions
   Creates visual continuity: PUBLIC copper invite → SUBMITTER copper submit

2. TYPOGRAPHY CONSISTENCY:
   Both use Manrope 800 for headers (dominance, authority)
   Both use Manrope 700 11px uppercase for labels (system identity)
   Both use Inter 400 for body (warmth, readability)

3. SPACING & RHYTHM:
   Both use 12px/16px/20px spacing grid
   Both use 8px gap increments for components
   Both use 12px border radius for cards (rounded, friendly)

4. BUTTON PROGRESSION:
   PUBLIC: Copper button = "Explore" / "Check Dates"
   SUBMITTER: Copper button = "Continue" / "Submit Booking"
   The color creates visual momentum from discovery → request

5. SHARED DESTINATION:
   Both surfaces lead to "My Bookings/My Properties" destination
   Same header style, same card layout, same status pills
   Proves they're part of same system, not separate products

6. PROPERTY CONTEXT:
   PUBLIC shows property in cards across listing
   SUBMITTER shows property in context bar at top of every form
   This creates narrative continuity: "I'm booking THIS property"
