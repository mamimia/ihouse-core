DESIGN NOTES & EXPERT CONSTRAINTS
Role Personality
Priority-first, issue-driven. Maintenance work is not time-scheduled — it is severity-ranked by CRITICAL → HIGH → NORMAL. Cards sort by priority, never by time. The SLA pill shows response deadline, not checkout time. Vocabulary: Job, Issue, Work Order — never Task or Clean.
Access Code Always Visible
Every job card and detail screen prominently displays the property access code. Maintenance workers arrive at locked properties — the access code is the most critical operational data they need immediately.
Distinct from Cleaner Role
Maintenance uses issue type tags (Plumbing, Electrical, Pool, AC, General, Gardening) not room lists. Progress is step-based (Assess → Document → Fix → Test → Confirm), not room-by-room. Before/after photo pair required. Can't Complete path always visible for blockers.
Future Specialty Support
The issue type tag system supports future sub-specialties. Each specialty has its own types and skill tags on Profile. Generalists see all jobs; specialists see filtered by their skill set. Profile shows maintenance-specific skill tags.
