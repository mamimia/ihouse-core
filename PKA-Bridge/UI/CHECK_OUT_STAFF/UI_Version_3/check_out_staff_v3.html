Check-Out Staff

Operational UI Direction — Version 3

Role Personality Closure-driven, verification authority, settlement engine operator. Staff verifies departures, inspects property condition, resolves meter readings, and finalizes guest refunds. Departure-only scope: sees only this booking's data, no cross-property or worker visibility.
V3 Changes from V2 Expanded from 3-screen overview to 6-screen prototype showing full 5-step wizard architecture. Added Meter OCR with live delta preview (opening + usage + estimated charge). Added Settlement calculation with deposit/electricity/damage cards. Added irreversibility warning on final submit. Added turnover chain visualization showing CHECK-OUT → CLEANING → INSPECTION → READY → CHECK-IN sequence. Aligned to settlement engine phases 959–961 and task state machine PENDING → ACKNOWLEDGED → IN_PROGRESS → COMPLETED.
Key Patterns Screen 1: Departure queue with overdue pulse animation, day health strip, status pills. Screen 2: 3-tab photo comparison (Reference | Check-in | Checkout) with room selector and inspection notes toggle. Screen 3: OCR meter capture with live delta preview. Screen 4: Settlement cards (deposit, electricity, issues) with deposit resolution radio options. Screen 5: Final summary with settlement breakdown and irreversibility warning. Screen 6: Success with turnover chain and auto-created cleaning task.
Shaped By Claudia (property inspection standards, room-by-room comparison, damage classification). Marco (dark theme, touch-friendly photo tabs, field conditions). Sonia (departure-only scope, no cross-worker or financial aggregate visibility). Talia (settlement clarity, deposit resolution options, All Good vs Issues Found prominence). Hana (turnover chain awareness, cleaning auto-trigger).
