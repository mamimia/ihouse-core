Maintenance — Operational UI Direction
Domaniqo · March 2026 · 4 screens: Home · Job Detail · Job Execution · Profile
Role personality
Priority-first, issue-driven. Maintenance work is not time-scheduled — it is severity-ranked. Cards sort by CRITICAL → HIGH → NORMAL, not by arrival time. The SLA pill shows response deadline (not checkout time). Vocabulary: Job, Issue, Work Order — never "Task" or "Clean".
Access code always visible
Every job card and job detail screen shows the property access code prominently. Maintenance workers arrive at locked properties — the access code is the most critical operational data they need. It appears in a dark pill on the card and as a full-width block in job detail.
Distinct from Cleaner
Maintenance uses issue type tags (Plumbing, Electrical, Pool, AC…) not room lists. Progress is step-based (Assess → Document → Fix → Test → Confirm), not room-by-room. Before/after photo pair required. Can't Complete path always visible — maintenance can hit blockers that cleaners don't.
Future specialty support
The issue type tag system is designed to support future sub-specialties (Electrical, Plumbing, Pool, Gardening). Each specialty gets its own issue types and skill tags on Profile. The generalist worker sees all jobs. Specialists see only their category. Same UI, filtered scope.
